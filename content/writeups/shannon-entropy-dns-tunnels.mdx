DNS makes a great covert channel because it's nearly always allowed outbound. The
defender's problem is that the bytes look like names, and names look like anything.

## The signal

Human-friendly domains cluster around low-to-mid Shannon entropy. Encoded payloads —
base32, base64-ish — sit much higher because their character distribution is closer to
uniform. That gap is the entire opportunity.

```python title="entropy.py"
def entropy(s: str) -> float:
    n = len(s) or 1
    return -sum((c / n) * math.log2(c / n) for c in Counter(s).values())
```

Run that over the leftmost label of every query and you already have a usable ranking
signal — high scores float covert channels toward the top of the list.

## Where it breaks

Entropy on its own is not a detector; it's one weak signal. The failure modes are
predictable:

- CDNs and analytics platforms emit high-entropy hostnames legitimately.
- Short labels make entropy noisy — combine it with length and cadence.
- Attackers can throttle and pad to dodge naive thresholds.

The fix is to **stop treating entropy as a threshold and start treating it as a feature** —
one input to a weighted score alongside label length and per-source query rate. That's
exactly what [DNS Exfil Sentinel](/projects/dns-exfil-sentinel) does.
