"Just an endpoint that runs nmap" is one of those requests that sounds trivial and is
actually a security review waiting to happen. The honest version has more moving parts than
anyone expects.

## Never touch the shell

The single most important design rule: user input never becomes a shell string. Zod
validates every field, and the CLI is invoked with an argv array against an allowlist of
flags.

```typescript title="routes/scan.ts" {1,11}
const ScanInput = z.object({
  target: z.string().regex(CIDR_OR_HOST),
  profile: z.enum(["quick", "service", "vuln"]),
  ports: z.string().regex(/^[0-9,\-]+$/).optional(),
});

app.post("/scans", { preHandler: requireScope("scan:write") }, async (req) => {
  const input = ScanInput.parse(req.body);
  const job = await scanQueue.add("nmap", input, {
    attempts: 1,
    removeOnComplete: 200,
  });
  return { id: job.id, status: "queued" };
});
```

The two highlighted lines are the whole safety story: a strict input schema at the top, and
a queued job (never a synchronous spawn) at the bottom. Everything between them is plumbing.

## Backpressure matters

The queue isn't just tidiness — nmap is resource-hungry, and per-tenant concurrency budgets
are what keep one noisy client from starving everyone else on a small node.
